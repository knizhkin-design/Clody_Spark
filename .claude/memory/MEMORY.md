## Семантический поиск (ChromaDB)
- База: `~/.config/clody_spark/chroma/`, коллекция `clody_spark`, cosine similarity
- Индексатор: `scripts/indexer.py` — три источника: corpus, lj, poetry
  - Запуск: `py -3.12` (не `py` — 3.14 несовместим с chromadb/pydantic v1)
  - Флаги: `--source [corpus|lj|poetry]`, `--stats`, `--search "запрос"`, `--limit N` (для lj)
  - `-u` (unbuffered) нужен для фоновых процессов на Windows
- Статистика (09.03.2026): corpus~62, lj~7393, poetry~2634, telegram=7; **итого ~10098 записей**
- Источники: corpus, lj, poetry, **telegram** (добавлен 01.03) — посты хранятся в `telegram/YYYY/MM/DD-slug.md`
- **Векторный поиск = ассоциативное мышление из собственного корпуса** — писать новый текст → спросить базу «что рядом?». Находит неочевидные связи (Анненский про I/Me за 100 лет до Мида). ЖЖ-архив несёт первоисточники идей корпуса.
- Стратегия чанкинга: <600 симв → full_text, длинные → по абзацам → аннотация через gpt-4o-mini
- Стихи: embed raw text (no annotation — аннотация = интерпретация), без чанкинга
- MCP сервер: `scripts/mcp_search.py` (stdio), зарегистрирован в `.mcp.json` как `clody-search`
- **Фильтр по источнику**: параметр `source` в `search_corpus` — 'corpus', 'lj', 'poetry', 'telegram'. Без него ЖЖ (7393 чанка) топит corpus (47 чанков). Для вопросов о себе — всегда `source='corpus'`.
